# Enterprise Deployment

This guide shows how to install the add-in for multiple users at once in your organization. It follows the official
[Microsoft installation guide](https://learn.microsoft.com/en-us/microsoft-365/admin/manage/office-addins?view=o365-worldwide#upload-custom-office-add-ins-in-your-organization).

### Before You Start

You will need:

- The so-called `manifest` file: <a href="../../static/cloud.xml" download="singledraft-manifest.xml">download manifest</a>
- Microsoft Office 365 account with one of these admin roles:
  - Global Administrator
  - Azure Application Administrator
  - Exchange Administrator
- Users must have one of these license types:
  - Microsoft 365 Business (Basic, Standard, or Premium)
  - Office 365 Enterprise (E1, E3, E5, or F3)
  - Microsoft 365 Enterprise (E3, E5, or F3)

For complete requirements, see the [Microsoft documentation](https://learn.microsoft.com/en-us/microsoft-365/admin/manage/centralized-deployment-of-add-ins?view=o365-worldwide).

### Step-by-Step Installation

- **Go To Microsoft Office 365** - On the left sidebar click `Admin` icon.<br/>

  <Image img={step1} style={{ width: 200 }} />

- **Go to Settings > Integrated apps.**<br/>

  <Image img={step2} style={{ width: 200 }} />

- **Click on Upload custom apps.**<br/>

  <Image img={step3} style={{ width: 300 }} />

- **Select App type = Office Add-in**<br/>

  <Image img={step4} style={{ width: 300 }} />

- **Upload manifest XML** - Choose `Upload manifest file from device` and use the file provided above<br/>

  <Image img={step5} style={{ width: 400 }} />

- **Click Next and Add users**: - Keep the toggle for test deployment to `No` - assign which users should be able to see and use the Add-in<br/>

  <Image img={step6} style={{ width: 400 }} />

- **Finish deployment**

  - Click Next
  - Accept permission request by clicking Next
  - Click Finish deployment

  accounts that were assigned in previous step.
